# iOS: convert from Swift Package Manager to CocoaPods

## Why

The iOS project was scaffolded in **Swift Package Manager** mode (`ios/App/CapApp-SPM`,
no `Podfile`). But `@capacitor-firebase/authentication@6.3.1` ships **only a
CocoaPods podspec** (`CapacitorFirebaseAuthentication.podspec`) — it has **no
`Package.swift`** — so SPM cannot resolve it and the build fails with:

```
the package manifest at '.../@capacitor-firebase/authentication/Package.swift'
cannot be accessed ... doesn't exist in file system
```

The committed `ios/App/CapApp-SPM/Package.swift` is also broken: it has Windows
backslash paths and references `CapacitorPreferences` (a plugin the app doesn't
use). It was generated by `cap sync` on the Windows dev machine and never
worked on macOS.

Fix: switch the iOS platform to CocoaPods, which every `@capacitor-firebase`
plugin officially supports. **This must be done once on a Mac** (needs Xcode +
CocoaPods), then committed. After that, CI just runs `pod install` + build.

## Steps (on a Mac)

```bash
cd mobile
pnpm install

# 1. Regenerate the iOS platform in CocoaPods mode (Capacitor 6 default).
#    Back up any native customizations first (see step 4).
rm -rf ios
pnpm exec cap add ios          # creates ios/ with a Podfile (no CapApp-SPM)

# 2. Build the web assets and sync (installs pods, wires plugins into the Podfile).
pnpm build
pnpm exec cap sync ios

# 3. Confirm it compiles unsigned for the simulator.
cd ios/App
pod install
xcodebuild build \
  -workspace App.xcworkspace \
  -scheme App \
  -configuration Debug \
  -sdk iphonesimulator \
  -destination 'generic/platform=iOS Simulator' \
  CODE_SIGNING_ALLOWED=NO
```

## 4. Re-apply native config lost by the regen

`cap add ios` recreates `ios/` from the template, so re-apply anything custom
that was in the old project:

- **`GoogleService-Info.plist`** — download from the Firebase Console for the
  `com.arohaastrology.app` iOS app and add it to the `App` target in Xcode
  (Copy items if needed → App target). See `ios/README-firebase-setup.md`.
- **`Info.plist`** — re-check any custom keys (e.g. camera/notification usage
  strings, URL schemes) against the old `ios/App/App/Info.plist` in git history:
  `git show HEAD:ios/App/App/Info.plist`.
- `contentInset: 'always'` and other `ios` options in `capacitor.config.ts` are
  re-applied automatically by `cap sync`.

Review the full diff before committing: `git status`, `git diff`.

## 5. Update CI

Once the CocoaPods project is committed, update
`.github/workflows/ios-build.yml` to build the workspace with pods:

```yaml
      - run: pnpm exec cap sync ios
      - name: pod install
        working-directory: ios/App
        run: pod install
      - name: Build for simulator (no code signing)
        working-directory: ios/App
        run: |
          xcodebuild build \
            -workspace App.xcworkspace \
            -scheme App \
            -configuration Debug \
            -sdk iphonesimulator \
            -destination 'generic/platform=iOS Simulator' \
            CODE_SIGNING_ALLOWED=NO
```
